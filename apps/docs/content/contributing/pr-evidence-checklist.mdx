# PR evidence checklist

Use this checklist before you request review on a tax behaviour or public
contract change.

## Required evidence

- Source citations for every changed fact, rule, parameter table, calculator
  or tax year.
- Owning package for each changed schema, type, branded ID, service tag, layer
  and tagged error.
- Golden tests or regression tests for changed tax results.
- Type tests for public SDK input, output or descriptor changes.
- API compatibility tests for HTTP request, response, error or metadata
  changes.
- SDK compatibility tests for SDK helper or schema export changes.
- Graph, trace or ledger evidence when rule dependencies or outputs change.
- Release-impact notes that say whether behaviour is patch, minor, major or
  docs-only.
- Changeset path, or a no-Changeset rationale for docs-only or app-internal
  work.
- `bun run verification` result.
- `bun run changeset status --verbose` result.

## Evidence format

```md
### Source citations

- ATO, Schedule 1, table 2, 2025-26, accessed 2026-06-01.

### Tests

- `bun run --filter=@taxkit/rules-au-pay test`
- `bun run --filter=@taxkit/sdk test-types`
- `bun run verification`

### Compatibility

- API compatibility tests: passed.
- SDK compatibility tests: passed.
- Generated OpenAPI: reviewed for changed `CalculatorRunRequest` fields.

### Release impact

- Changeset: `.changeset/example.md`
```

## Review path

Reviewers will compare your evidence with
[Review expectations](./review-expectations.mdx), the owning package README
and the linked architecture docs.
