# Build a server route with the SDK

Use this guide when your application owns the HTTP route and only needs
TaxKit for the calculation.

## Before you start

- Use a server-side TypeScript runtime.
- Install the SDK with [Install the SDK](../start/install-the-sdk.mdx).
- Keep request validation at your application boundary.

## Steps

1. Parse your application request.
2. Convert accepted fields into canonical TaxKit facts.
3. Call the SDK helper.
4. Return the report fields your application needs.

## Example

```ts
import { aud } from "@taxkit/core/primitives";
import { GrossPay } from "@taxkit/rules-au-pay";
import { au } from "@taxkit/sdk/au";

export const handlePayPreview = async (request: Request): Promise<Response> => {
  const body = (await request.json()) as {
    grossPayCents: number;
    period: "weekly" | "fortnightly" | "monthly";
    taxFreeThresholdClaimed: boolean;
  };

  const report = await au.pay.takeHomePay({
    grossPay: new GrossPay({
      amount: aud(body.grossPayCents),
      period: body.period,
    }),
    taxFreeThresholdClaimed: body.taxFreeThresholdClaimed,
  });

  return Response.json({
    netPayCents: report.netPay.cents,
    withholdingsCents: report.withholdingsTotal.cents,
  });
};
```

## Verify the result

Request body:

```json
{
  "grossPayCents": 165400,
  "period": "weekly",
  "taxFreeThresholdClaimed": true
}
```

Expected response:

```json
{ "netPayCents": 130100, "withholdingsCents": 35300 }
```

## Handle errors

Use `TaxKit.safe.calculate` if your route should return expected failures
without a thrown Promise rejection. Map the safe failure to your application's
HTTP status and response shape.
