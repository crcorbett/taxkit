# Schemas

Use SDK schema exports when your application needs the canonical calculator run
contracts or safe-result error shapes.

## Import schemas

```ts
import {
  CalculatorRunFacts,
  CalculatorRunRequest,
  CalculatorRunResponse,
  CalculatorRunServiceRequest,
  CalculatorServiceError,
} from "@taxkit/sdk/schemas";
```

These names are re-exported from the calculator package for consumer
convenience. The calculator package remains the source of truth for the
`CalculatorRun*` contracts and `CalculatorServiceError`.

## Common contracts

| Name                          | Use                                           |
| ----------------------------- | --------------------------------------------- |
| `CalculatorRunFacts`          | Canonical facts accepted by calculator runs.  |
| `CalculatorRunRequest`        | Public calculator run payload shape.          |
| `CalculatorRunServiceRequest` | Service-level request with the calculator id. |
| `CalculatorRunResponse`       | Canonical run response shape.                 |
| `CalculatorServiceError`      | Expected calculator service failure union.    |

## Safe-result schemas

The SDK also exports safe-result and error schemas:

```ts
import {
  TaxKitFailure,
  TaxKitSchemaDecodeError,
  TaxKitSuccess,
} from "@taxkit/sdk/schemas";
```

Use these when you persist, inspect or validate results returned by
`TaxKit.safe.calculate`.
