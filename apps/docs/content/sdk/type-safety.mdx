# Type safety

WhatTax uses TypeScript types and runtime schemas together. TypeScript checks
the calculator descriptor and facts you pass at compile time. Runtime schema
decoding still validates untrusted values before calculation.

## Compile-time calculator inputs

The selected descriptor controls the accepted facts and the returned report:

```ts
import { aud } from "@whattax/core/primitives";
import { WhatTax } from "@whattax/sdk";
import { au } from "@whattax/sdk/au";

const report = await WhatTax.calculate(au.calculations.annualIncomeTax, {
  taxableIncome: aud(9_000_000),
});

report.liability.cents;
```

If you pass facts for another calculator, the call should fail type checking:

```ts
import { aud } from "@whattax/core/primitives";
import { WhatTax } from "@whattax/sdk";
import { au } from "@whattax/sdk/au";

await WhatTax.calculate(au.calculations.annualIncomeTax, {
  grossPay: aud(165_400),
  taxFreeThresholdClaimed: true,
});
```

## Branded values

Money values use canonical branded constructors from `@whattax/core`:

```ts
import { aud } from "@whattax/core/primitives";

const taxableIncome = aud(9_000_000);
```

Use these constructors instead of raw numbers when a calculator fact expects a
branded value.

## Runtime schema decoding

Compile-time checks protect code you compile. Runtime schema decoding protects
values that come from users, forms, JSON or other services.

This illustrative block shows a value runtime decoding rejects. It is not
intended to typecheck, because TypeScript also catches the wrong input shape.

```ts
import { WhatTax } from "@whattax/sdk";
import { au } from "@whattax/sdk/au";

const result = await WhatTax.safe.calculate(au.calculations.annualIncomeTax, {
  taxableIncome: "90000",
});

if (result._tag === "WhatTaxFailure") {
  console.error(result.error._tag);
}
```

The safe facade returns a typed failure value for expected decode or calculation
failures.

## Full run typing

Effect consumers can preserve the full `CalculatorRunResponse` shape while
narrowing `report` to the selected descriptor output:

```ts
import type { SdkCalculatorRunResponse } from "@whattax/sdk/effect";
import type { CalculatorRunResponse } from "@whattax/sdk/schemas";
```

`SdkCalculatorRunResponse<Report>` keeps the canonical response shape and
narrows `report` to the decoded report type.
