# OpenAPI reference

Use generated OpenAPI for endpoint paths, parameters, request bodies and
response schemas. Narrative API pages explain when to use HTTP, how errors
work and how the API relates to the SDK.

## Generated source

The current API package generates the OpenAPI document from
`@taxkit/api-http` route definitions. The API app serves it at:

```txt
GET /api/docs/openapi.json
```

The local Scalar viewer is served at:

```txt
GET /api/docs
```

## Smoke check

Run the API locally, then fetch the generated document:

```sh
bun run --filter=api start
curl http://127.0.0.1:4000/api/docs/openapi.json
```

The document should include the calculate path and `CalculatorRunRequest`
payload schema. If a future docs app embeds generated reference pages, the
embed should consume this generated document rather than copying endpoint
fields by hand.

```txt
/api/v1/calculators/{calculatorId}/calculate
CalculatorRunRequest
```

## Accepted exclusion

The docs app does not fetch live OpenAPI JSON during `docs` or
`@taxkit/docs-content` builds. That would make static docs validation depend
on a running API process. For this slice, the accepted exclusion is a static
reference smoke in docs-content validation: the page must name
`/api/docs/openapi.json`, the calculate path and the generated request schema.

A later reference slice can add an offline generated OpenAPI artifact or a
live integration smoke that starts `apps/api` explicitly.
